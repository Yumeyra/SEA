1. Каква функционалност е реализирана
В рамките на този етап беше реализиран основен функционален модул от системата EventAura, свързан с управлението и визуализирането на събития, както и закупуването на билети. Реализираните компоненти включват:

Страница за детайли на събитие
Добавяне на билети в количката
Финализиране на поръчка
Система за „Любими“
Филтриране на събития по град, категория и дата
Падащо меню за градове 
Тъмен и светъл режим с пълна поддръжка
Оптимизиран и рефакториран CSS за по‑добра четимост и UX
Това представлява напълно работещ функционален пакет, който е ключов за основната цел на платформата — продажба на билети.

2. Как работи
Основен поток на работа:
Потребителят отваря списъка със събития.
Може да филтрира по град, категория или дата.
При избор на събитие се зарежда страницата с детайли:
плакат ,описание ,дата, място, цена ,оставащи места

Регистриран потребител може да:

избере количество

добави в количката

финализира покупката

получи билет в профила си

Потребителят може да добавя събития в „Любими“.

Администраторът може да управлява събитията през административния панел.

3. Какви проблеми възникнаха

Проблем 1: Тъмният режим не работеше коректно
Част от текстовете ставаха невидими.
Bootstrap override‑ите се сблъскваха с персонализирания CSS.

Проблем 2: Падащото меню за градове беше „сбито“
Липсваше spacing.
Старият работещ CSS беше коментиран и не се прилагаше.

Проблем 3: Детайлите за събитие не се стилизираха правилно
HTML структурата използваше Bootstrap класове, а CSS таргетираше други класове.
Това водеше до липса на стилове в тъмен режим.

Проблем 4: Проблеми с валидирането на формите
При добавяне на събитие:
някои полета не се валидираха правилно
грешките не се визуализираха
имаше проблеми с датата и цената

4. Как бяха решени
Решение на проблем 1:
Премахнати конфликтни правила.
Добавени коректни CSS променливи за тъмен режим.
Изчистени дублиращи се стилове.

Решение на проблем 2:
Възстановен оригиналният работещ CSS за mega‑menu.
Добавени spacing, padding и hover ефекти.
Добавена поддръжка за тъмен режим.

Решение на проблем 3:
Анализирана HTML структурата.
Създадени точни CSS селектори за .event-info-label и .event-info-value.
Добавен финален fix за тъмен режим.

Решение на проблем 4:Добавяне на атрибути за валидация в моделите  
Бяха използвани стандартни ASP.NET Core атрибути като
Required, StringLength, Range, EmailAddress и др.,
за да се гарантира, че всички задължителни полета се проверяват още на ниво модел.
Активиране на клиентска валидация  
Уверих се, че в страниците са включени необходимите JavaScript библиотеки за unobtrusive validation, което позволи грешките да се показват веднага, без презареждане на страницата.
Коригиране на Razor формите  
Някои полета не визуализираха грешки, защото липсваха asp-validation-for елементи.
След добавянето им грешките започнаха да се показват правилно под всяко поле.
Проверки в контролерите  
Добавена беше проверка ModelState.IsValid преди обработка на данните, за да се предотврати записване на невалидна информация в базата.
Тестване с различни сценарии  
Формите бяха тествани с празни полета, грешни формати, твърде дълги стойности и невалидни дати, за да се гарантира, че валидацията работи коректно във всички случаи.
